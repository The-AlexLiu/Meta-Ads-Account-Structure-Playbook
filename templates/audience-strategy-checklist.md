# Meta Ads 受众策略检查清单

用于判断一个账户的受众设置是在帮助系统学习，还是在限制系统学习。

## 1. 受众边界

| 检查项 | 状态 | 备注 |
|---|---|---|
| 投放国家和地区明确 | 待检查 |  |
| 语言设置符合落地页和素材语言 | 待检查 |  |
| 年龄设置符合产品和合规要求 | 待检查 |  |
| 特殊广告类别限制已确认 | 待检查 |  |
| Purchaser / 内部人员 / 测试用户排除逻辑清楚 | 待检查 |  |

## 2. 受众信号

| 受众类型 | 用途 | 当前是否使用 | 判断 |
|---|---|---|---|
| Broad / Advantage+ Audience | 主力放量 | 待检查 | 是否给系统足够学习空间 |
| Interest | 诊断场景或购买理由 | 待检查 | 是否只是辅助测试，而不是主结构 |
| Lookalike | 冷启动或扩量提示 | 待检查 | seed 是否足够高质量 |
| Custom Audience | 排除、再营销、系统提示 | 待检查 | 人群是否足够大且新鲜 |
| Retargeting | 高意向承接 | 待检查 | 是否有频次和增量判断 |

## 3. 风险检查

| 风险 | 判断标准 | 结论 |
|---|---|---|
| 受众拆太细 | 多个 ad set 分别只有很小预算 | 待判断 |
| 兴趣过度堆叠 | 同一产品同一 offer 拆太多兴趣 | 待判断 |
| Lookalike seed 质量差 | seed 太小、太旧或包含低质量用户 | 待判断 |
| 再营销虚高 | ROAS 高但整体 GMV 或新客没有增长 | 待判断 |
| 过度排除 | 排除太多导致学习空间不足 | 待判断 |
| controls / suggestions 混淆 | 把建议受众误认为硬定向 | 待判断 |

## 4. 结论

| 问题 | 判断 |
|---|---|
| 主力 campaign 是否适合 Broad / Advantage+ Audience | 待判断 |
| 是否需要单独兴趣测试 | 待判断 |
| 是否需要 Lookalike suggestion | 待判断 |
| 是否需要 Retargeting campaign | 待判断 |
| 是否需要排除 Purchaser | 待判断 |
